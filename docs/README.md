# Documentation

This directory contains the documentation for the RAG Document Assistant project.

## Structure

- `source/` - Source files for Sphinx documentation
- `build/` - Built documentation (generated by Sphinx)

## Documentation Map

| Document | Purpose | Primary Audience |

| [Architecture](architecture.md) | Technical system design and components | Developers, Architects |
| [Implementation Guide](implement.md) | Code organization and implementation details | Developers |

| [Test Results](test_results.md) | Detailed testing outcomes and metrics | QA Engineers, Developers |
| [Case Study](case_study.md) | Comprehensive technical case study | Architects, Technical Leads |

## Building Documentation

To build the documentation locally:

```bash
# Install Sphinx
pip install sphinx sphinx-rtd-theme

# Build HTML documentation
cd docs
make html

# Open the documentation
open build/html/index.html
```

## Contributing to Documentation

To contribute to the documentation:

1. Edit the files in `source/`
2. Build the documentation locally to verify changes
3. Submit a pull request with your changes

The documentation uses reStructuredText format. See the [Sphinx documentation](https://www.sphinx-doc.org/en/master/) for syntax details.
